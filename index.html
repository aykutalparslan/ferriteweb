Open Source Telegram Server written in C#



On-Premises or in the Cloud

Complements the open source Telegram ecosystem so that you can host your own Telegram Service.

Pluggable Storage

Ferrite supports pluggable storage engines so that you can choose the storage engine depending on your scale.

Fits Right In

Original Telegram applications are not modified other than replacing the server IP and the encryption keys.

Active Development

The project is actively maintained and strives through being a feature complete Telegram Server with high performance.

Behind the Scenes

From a single container to a large cluster

Ferrite is designed to run in diverse scenarios hence it supports RocksDB and in-process data structures together with embedded Lucene as well as Cassandra, Redis, Kafka and MinIO. Depending on your use case you can start with a single container or choose to manage massive clusters of services. It is also relatively easy to add new data stores if you choose to do so.
